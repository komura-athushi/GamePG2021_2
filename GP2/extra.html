  • 1. 変数

    • 1.1 命名

      • 突然だが、以下のコードの意味が分かるだろうか？

      F.cpp 

      
                
      
                    
F::G()
{
        
    if(b == 0)
    {
        a.y += c;
        if(a.y >= d)
        {
            b = 1;
        }
    }
    else
    {
        a.y -= c;
        if(a.y <= -d)
        {
            b = 0;
        }
    }

    e.SetPosition(a);
}

      これはオブジェクトを上下に動かすコードの一例だが、なかなか分かりづらいと思う。
      では、なぜ分かりにくかったのだろうか？
      答えは、変数名が「a」や「b」などいい加減だったからである。 また、クラス名も「G」、関数名も「G」でどういうクラスなのか、どういう関数なのか分からない。
      では、変数名、クラス名、関数名をしっかりしたものにしてみよう。

      Box.cpp 

      
                
      
                    
Box::Move()
{      
    if(moveState == 0)
    {
        position.y += MOVE_SPEED;
        if(position.y >= LIMIT)
        {
            moveState = 1;
        }
    }
    else
    {
        position.y -= MOVE_SPEED;
        if(position.y <= -LIMIT)
        {
            moveState = 0;
        }
    }
                        
    modelRender.SetPosition(position);
}

      変数名、クラス名、関数名をしっかりしたものに変更したことで、コードが格段に見やすくなったのではないだろうか？
      このように、命名をしっかりすることはコードを綺麗にする第一歩である。
